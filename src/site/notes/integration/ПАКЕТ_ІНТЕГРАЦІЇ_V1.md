## 0. Огляд

Три компоненти системи, кожен з окремим integration bundle:

| Компонент | Технологія | Порт | Роль |
|-----------|-----------|------|------|
| **Lovable Frontend** | React SPA (Vite + Tailwind) | — | Projection layer; read + create tasks |
| **BLOOM Runtime** (Replit) | Express + PostgreSQL | 5000 | Orchestrator; task queue; proxy |
| **Node Agent** (Membridge) | FastAPI (Python) | 8001 | Worker; Claude CLI execution |

**Ваша інфраструктура:**
- Alpine x86 — BLOOM Runtime (Express :5000) + Membridge CP (:8000) + Agent (:8001)
- Cloudflare Tunnel — **bloom.exodus.pp.ua** → Alpine :5000 (зовнішній доступ)
- Raspberry Pi 3B — Agent (:8001)
- Orange Pi PC2 — Agent (:8001)
- MinIO — на Alpine (apiminio.exodus.pp.ua)

**E2E статус (2026-02-27):** ✅ Health → Worker online → Task create → Dispatch → Heartbeat (10с) → Poll (running → completed) → Artifact ("HELLO BLOOM")

## A. Що дати Lovable (Frontend)

### A.1 Принципи

З `ІНТЕГРАЦІЯ_FRONTEND_LOVABLE` (Axiom A6) та `LOVABLE_INITIAL_INSTRUCTION §3`:

- **Projection only:** Frontend читає стан, не має write authority до storage
- **Без секретів:** Жодних API keys, admin keys, MinIO credentials у коді
- **Через backend:** Усі API calls → Express на порту 5000
- **Polling:** Немає WebSocket/SSE у V1; polling з адаптивними інтервалами
- **Два шари пам'яті не змішувати:** claude-mem.db ≠ DiffMem/git
- **Finalized artifacts immutable:** UI не пропонує редагування
- **JWT:** httpOnly cookie або sessionStorage (**НЕ** localStorage)

### C.5 /execute-task Contract

**Request (DispatchEnvelope):**
```json
{
  "task_id": "uuid",
  "prompt": "Проаналізуй...",
  "context_id": "garden-seedling",
  "agent_slug": "writer",
  "desired_format": "markdown",
  "context_hints": ["docs/*.md"],
  "policy": { "timeout_sec": 120, "budget": { "max_tokens": 0 } },
  "runtime_url": "http://RUNTIME:5000"
}
```

**Поведінка Node:**
1. Validate envelope
2. Return immediate response: `{ "ok": true, "started": true }`
3. Start heartbeat thread: `POST runtime_url/api/runtime/llm-tasks/{task_id}/heartbeat` кожні 10с
4. Execute: `claude --prompt "..." --format markdown`
5. On completion: `POST runtime_url/api/runtime/llm-tasks/{task_id}/complete`
6. Stop heartbeat thread

## D. Тестування End-to-End

### D.1 Smoke Test

```bash
# 1. Health check Runtime
curl -s http://RUNTIME:5000/api/runtime/health | jq .status
# Expected: "ok"

# 2. Перевірити workers
curl -s http://RUNTIME:5000/api/runtime/workers | jq '.[].status'
# Expected: "online" для кожної ноди

# 3. Створити задачу
TASK_ID=$(curl -s -X POST http://RUNTIME:5000/api/runtime/llm-tasks \
  -H "Content-Type: application/json" \
  -d '{"context_id":"test","agent_slug":"writer","prompt":"Say hello","desired_format":"text"}' \
  | jq -r .id)

# 4. Dispatch
curl -s -X POST http://RUNTIME:5000/api/runtime/llm-tasks/$TASK_ID/dispatch | jq .dispatched
# Expected: true

# 5. Чекати completion (poll)
for i in $(seq 1 30); do
  STATUS=$(curl -s http://RUNTIME:5000/api/runtime/llm-tasks/$TASK_ID | jq -r .status)
  echo "Attempt $i: $STATUS"
  [ "$STATUS" = "completed" ] && break
  sleep 5
done

# 6. Перевірити artifact
curl -s http://RUNTIME:5000/api/runtime/artifacts?task_id=$TASK_ID | jq '.[0].type'
```
