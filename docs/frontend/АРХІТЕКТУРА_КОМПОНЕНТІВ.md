# Компонентна архітектура Frontend

> **Для кого:** Claude (logic/type review), Lovable (UI generation), будь-який розробник що торкається компонентів.

---

## Шари компонентів

```
pages/                     ← Route-level components (один файл = один маршрут)
  └── RuntimeOverview, NotePage, EditorPage, AgentsPage, ...

components/
  ├── garden/              ← Feature components (62 компоненти, Lovable-owned)
  ├── memory/              ← DiffMem temporal UI (Claude-owned)
  ├── notebooklm/          ← NotebookLM integration
  ├── runtime/             ← Runtime header/footer
  ├── zones/               ← Zone access management
  ├── ui/                  ← shadcn-ui base primitives (Radio, Button, Dialog...)
  ├── AccessGateUI.tsx     ← Standalone access control UI
  └── AccessGuard.tsx      ← Route wrapper that checks zone access
```

## Ownership rules (практичне)

**Lovable може без погодження:**
- Додавати/змінювати JSX в `components/garden/**` і `pages/**`
- Додавати нові shadcn-ui компоненти в `components/ui/`
- Змінювати `vite.config.ts`, `tailwind.config.js`

**Claude review required перед merge:**
- Будь-які зміни в `hooks/`, `lib/`, `types/`
- Будь-які зміни в `components/memory/`
- Зміни в `App.tsx`, `package.json`
- Нові `fetch()` виклики будь-де

**Shared (merge вимагає явного рішення):**
- `App.tsx` (routing + auth flow)
- `.env*` файли
