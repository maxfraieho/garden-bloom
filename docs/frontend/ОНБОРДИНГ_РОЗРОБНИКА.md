# Frontend — Онбординг розробника

> **Якщо ти вперше торкаєшся frontend цього проєкту — почни тут.**
>
> Мета: до кінця цього документа ти розумієш де що живе, хто що будує, і як зробити першу зміну без порушення архітектури.

---

## 1. Зрозумій контекст: що таке Garden Bloom

Garden Bloom — система виконання AI-агентів, де **людина залишається в контролі**.

- Фронтенд — це цифровий сад: нотатки, граф знань, runtime-дашборд.
- Бекенд — це Cloudflare Worker gateway + Mastra orchestration (Python/Node).
- Агенти **пропонують** дії. Людина **погоджує або відхиляє**.

**Product name:** Garden Bloom (не "digital garden", не "knowledge base").

---

## 2. Зроби клон і запусти

```bash
git clone https://github.com/maxfraieho/digital-garden-setup.git
cd digital-garden-setup
npm install
npm run dev
```

Після `npm run dev` відкрий `http://localhost:8080`.

**Перша перевірка:** тебе зустріне `OwnerSetupWizard` (якщо немає власника) або `RuntimeOverview` dashboard.

---

## 3. Зрозумій архітектурний інваріант

**Одне правило, яке не можна порушити:**

> Frontend НІКОЛИ не звертається до бекенду напряму.
> Єдиний канал: `mcpGatewayClient` → Cloudflare Worker (`https://garden-mcp.exodus.pp.ua`).

Весь HTTP-трафік — через `requestJson<T>()` у `src/lib/api/mcpGatewayClient.ts`.

Ніяких прямих `fetch()`. Ніяких `axios`. Тільки mcpGatewayClient.

**Verification:** `grep -r "fetch(" src/` — в результатах не повинно бути прямих API-запитів поза mcpGatewayClient.

## 4. Зрозумій ownership model

Хто що будує:

| Область | Хто |
|---------|-----|
| `src/pages/**`, `src/components/garden/**` | **Lovable** — JSX структура, UX layout, shadcn компоненти |
| `src/hooks/**`, `src/lib/**`, `src/types/**` | **Claude** — логіка, типи, валідація, API-клієнт |
| `src/components/memory/**` | **Claude** — DiffMem integration logic |
| Security, type safety, RLS | **Claude** |

**Якщо ти Claude:** не переписуй JSX структуру в garden/, якщо не попросили явно. Фокус — хуки та логіка.

**Якщо ти Lovable:** не змінюй hooks/, lib/, types/ без Claude ревю.

## 5. Зрозумій структуру src/

```
src/
├── App.tsx              ← routing + auth gate (точка входу)
├── main.tsx             ← ReactDOM + ThemeProvider
├── components/
│   ├── garden/          ← 63 компоненти (Lovable-owned)
│   ├── memory/          ← DiffMem temporal memory UI (Claude-owned)
│   ├── notebooklm/      ← NotebookLM integration
│   ├── runtime/         ← runtime header/footer
│   ├── zones/           ← zone access management
│   ├── ui/              ← shadcn-ui base primitives
│   ├── AccessGateUI.tsx ← access control UI
│   └── AccessGuard.tsx  ← route-level access guard
├── hooks/               ← 22 кастомних хуки (Claude-owned)
├── lib/
│   ├── api/             ← mcpGatewayClient + apiErrorStore
│   ├── notes/           ← noteLoader, searchResolver, tagResolver, linkGraph
│   ├── chat/, comments/, drakon/, export/, i18n/
│   └── utils.ts
├── types/               ← TypeScript interfaces (Claude-owned)
│   ├── mcpGateway.ts    ← Gateway error codes, API types
│   ├── agentMemory.ts, agentRegistry.ts, drakonwidget.d.ts
└── pages/               ← один файл на роут
```

## 6. Зрозумій auth flow

1. `useOwnerAuth` перевіряє чи є owner токен у localStorage
2. Якщо `isLoading=true` → спінер
3. Якщо `isInitialized=false` → `OwnerSetupWizard`
4. Якщо ок → BrowserRouter + AccessGuard → маршрути

`AccessGuard` контролює доступ до зон (zone-based access control).

Специфікація auth: [[BLOOM_AUTH_UI_СПЕЦИФІКАЦІЯ]]
