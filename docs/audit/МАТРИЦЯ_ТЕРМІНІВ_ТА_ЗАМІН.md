# Матриця термінів та замін: BLOOM Runtime

> Створено: 2026-02-24
> Автор: Технічний редактор
> Статус: Canonical

## 1. Канонічний словник BLOOM Runtime

### Core Identity

| # | Канонічний термін (EN) | Канонічний термін (UA) | Визначення |
|---|------------------------|------------------------|------------|
| 1 | **BLOOM Runtime** | **Середовище виконання BLOOM** | Execution environment для behavioral logic |
| 2 | **Execution Context** | **Контекст виконання** | Ізольований runtime scope, активований master-key |
| 3 | **Behavioral Definition** | **Визначення поведінки** | Markdown-файл, що визначає логіку агента (замість "нотатка") |
| 4 | **Execution Graph** | **Граф виконання** | Мережа зв'язків між behavioral definitions (замість "knowledge graph") |

### Execution Pipeline

| # | Канонічний термін (EN) | Канонічний термін (UA) | Визначення |
|---|------------------------|------------------------|------------|
| 5 | **Execution Protocol** | **Протокол виконання** | 11-стадійний lifecycle від Intent до Completion |
| 6 | **IR (Intermediate Representation)** | **Проміжне представлення** | JSON-формат скомпільованої behavioral logic для execution |
| 7 | **Job** | **Завдання** | Атомарна одиниця виконання в runtime |
| 8 | **Lease** | **Оренда** | Делегування Job агенту з обмеженим часом дії |
| 9 | **Artifact** | **Артефакт** | Immutable результат execution |
| 10 | **Proposal** | **Пропозиція** | Immutable запит на мутацію canonical storage |

### Memory & Persistence

| # | Канонічний термін (EN) | Канонічний термін (UA) | Визначення |
|---|------------------------|------------------------|------------|
| 11 | **DiffMem** | **DiffMem** | Canonical persistence layer BLOOM runtime (git-based) |
| 12 | **Membridge** | **Membridge** | Інфраструктура синхронізації execution memory між вузлами |
| 13 | **Canonical Storage** | **Канонічне сховище** | Єдине джерело правди для execution artifacts |
| 14 | **Memory Binding** | **Прив'язка пам'яті** | Зв'язок між Execution Context та memory scope |

### Orchestration & Agents

| # | Канонічний термін (EN) | Канонічний термін (UA) | Визначення |
|---|------------------------|------------------------|------------|
| 15 | **Worker** | **Воркер** | Daemon-процес, що виконує Jobs (executor node) |
| 16 | **Capability** | **Спроможність** | Оголошена здатність Worker (напр. `llm.claude_cli`, `node`) |
| 17 | **Heartbeat** | **Heartbeat** | Періодичний сигнал від Worker до Control Plane |
| 18 | **Delegation** | **Делегування** | Передача execution scope від runtime до агента |
| 19 | **Task** | **Задача** | Верхньорівневий запит на виконання (містить один або більше Jobs) |

### Infrastructure

| # | Канонічний термін (EN) | Канонічний термін (UA) | Визначення |
|---|------------------------|------------------------|------------|
| 20 | **Control Plane** | **Площина управління** | Сервіс реєстрації, маршрутизації та моніторингу Workers |
| 21 | **Audit Log** | **Аудит-лог** | Immutable журнал усіх execution events |
| 22 | **Replay** | **Повтор** | Відтворення execution з persisted memory state |
| 23 | **Dispatcher** | **Диспетчер** | Компонент маршрутизації Tasks до Workers |

### Compilation & Logic

| # | Канонічний термін (EN) | Канонічний термін (UA) | Визначення |
|---|------------------------|------------------------|------------|
| 24 | **DRAKON** | **DRAKON** | Canonical visual logic format для behavioral definitions |
| 25 | **Compilation** | **Компіляція** | Перетворення DRAKON → Pseudocode → IR |
| 26 | **Behavioral Logic** | **Поведінкова логіка** | Визначена послідовність дій агента |

### Access & UX

| # | Канонічний термін (EN) | Канонічний термін (UA) | Визначення |
|---|------------------------|------------------------|------------|
| 27 | **Activate environment** | **Активувати середовище** | Вхід у BLOOM Runtime (замість "login" / "увійти в сад") |
| 28 | **Deactivate environment** | **Деактивувати середовище** | Вихід з BLOOM Runtime (замість "logout") |
| 29 | **Runtime Overview** | **Огляд середовища** | Головна сторінка (замість "Home" / "Dashboard") |
| 30 | **Expansion** | **Розширення** | Додаткова спроможність або модуль BLOOM Runtime |
